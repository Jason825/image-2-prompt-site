# ImagePromptive PRD V1

## 1. 文档目标

这份文档用于明确 `Image 2.0` 提示词网站第一阶段产品的目标、范围、核心页面、关键功能和交付边界，作为后续设计、开发和内容准备的执行依据。

## 2. 产品背景

当前 AI 图片生成需求越来越强，但多数用户在实际使用中会遇到几个问题：

- 不知道该怎么写 prompt
- 能搜到很多 prompt，但难以判断哪些真正可用
- 缺少结构化整理，复制之后仍然不会改
- 不同场景缺少现成模板，生成效率低

因此，本项目希望做一个只服务 `Image 2.0` 的垂直提示词网站，用结构化案例帮助用户：

- 快速找到可用 prompt
- 直接复制复用
- 学会如何改写与扩展

## 3. 产品目标

### 3.1 第一阶段目标

在第一阶段，网站需要完成以下目标：

- 建立一个可浏览、可搜索、可筛选的 `Image 2.0` Prompt 内容库
- 让用户能快速查看效果图并复制 Prompt
- 建立统一的数据结构，为后续扩展工具功能打基础
- 通过高质量精选内容验证用户是否愿意持续使用

### 3.2 不在第一阶段解决的问题

第一阶段不追求：

- 站内完整出图
- 完整用户社区
- 团队协作
- 企业功能
- API 能力

## 4. 产品定位

### 一句话定位

一个专注于 `Image 2.0` 的提示词库与案例网站，帮助用户发现、复制、理解和优化高质量 Prompt。

### 产品形态

第一阶段产品形态为：

`提示词库 + 效果展示`

第二阶段再逐步加入：

`轻工具能力`

## 5. 目标用户

### 5.1 新手用户

特征：

- 刚开始使用 AI 图片生成
- 不熟悉 Prompt 写法
- 希望直接复制现成内容

核心诉求：

- 找到能直接用的 Prompt
- 看图判断效果

### 5.2 设计师 / 内容创作者

特征：

- 需要持续寻找灵感
- 对视觉风格和出图效率较敏感

核心诉求：

- 快速筛选高质量案例
- 找到适合某种风格或场景的 Prompt
- 对已有 Prompt 做二次改写

### 5.3 电商 / 营销用户

特征：

- 更看重出图效率和模板复用
- 需要大量产品图、海报图、社媒图

核心诉求：

- 按场景快速找到模板
- 直接复制并替换主体内容

## 6. 用户核心任务

用户来到网站后，核心任务主要有 4 类：

1. 浏览灵感
2. 搜索合适 Prompt
3. 复制 Prompt 去使用
4. 基于已有 Prompt 继续修改

第一阶段优先解决前 3 类任务。

## 7. 第一阶段功能范围

## 7.1 必做功能

### 7.1.1 首页

首页目标：

- 让用户快速理解网站是做什么的
- 让用户立刻开始搜索或浏览
- 展示精选内容和热门分类

首页模块建议：

- 顶部主标题与价值描述
- 搜索框
- 热门分类入口
- 精选 Prompt 卡片
- 专题入口
- 最近新增或热门内容

### 7.1.2 Prompt 列表页

页面目标：

- 支持按条件快速浏览 Prompt
- 帮助用户找到最适合当前场景的内容

功能要求：

- 卡片列表展示
- 展示缩略图、标题、标签、简短说明
- 支持按关键词搜索
- 支持按分类筛选
- 支持按标签筛选
- 支持按排序浏览

### 7.1.3 Prompt 详情页

页面目标：

- 让用户完整查看一个 Prompt 的结构化内容
- 支持一键复制

核心模块：

- 标题
- 效果图
- 主 Prompt
- Negative Prompt
- 参数信息
- 标签与分类
- 使用场景说明
- 补充备注
- 一键复制按钮
- 相关推荐

### 7.1.4 搜索与筛选

搜索目标：

- 允许用户用少量关键词快速找到可用内容

筛选维度建议：

- 分类
- 风格
- 用途
- 是否人物
- 是否产品图
- 难度等级

### 7.1.5 一键复制

功能目标：

- 让用户在最短路径内获得 Prompt

要求：

- 支持复制主 Prompt
- 后续可扩展为复制完整 Prompt 组合
- 复制成功要有明确反馈

### 7.1.6 内容管理准备

虽然 V1 未必做完整后台，但至少要保证：

- 数据格式统一
- 内容可批量导入
- 后续能方便扩展到 CMS 或数据库

## 7.2 可选功能

以下功能可以在 V1.1 或 V2 增加：

- 收藏
- 登录
- Prompt 改写
- Prompt 扩展
- 中英切换
- 前后对比展示

## 7.3 不做功能

V1 不做：

- 登录后个人工作区
- 社区评论
- 在线生成图片
- 团队协作
- 支付系统深度接入

## 8. 页面结构

## 8.1 首页

主要信息结构：

- 顶部导航
- Hero 区
- 搜索入口
- 热门分类
- 精选 Prompt
- 专题推荐
- 页脚说明

### 首页核心 CTA

- 浏览 Prompt
- 按分类查看
- 搜索关键词

## 8.2 Prompt 列表页

信息结构：

- 页面标题
- 搜索输入框
- 分类筛选区
- 标签筛选区
- Prompt 卡片列表
- 分页或加载更多

## 8.3 Prompt 详情页

信息结构：

- 标题区
- 效果图展示区
- Prompt 内容区
- 参数与标签区
- 说明区
- 相关推荐区

## 9. 数据模型要求

每条 Prompt 至少包含以下字段：

- `id`
- `title`
- `slug`
- `model`
- `category`
- `tags`
- `prompt`
- `negative_prompt`
- `parameters`
- `preview_image`
- `notes`
- `source_name`
- `source_url`
- `license`
- `risk_level`
- `status`

建议预留字段：

- `revised_prompt`
- `lang`
- `difficulty`
- `is_featured`
- `published_at`

## 10. 内容策略

## 10.1 第一批内容建议

建议优先做 5 到 10 个高价值栏目，每个栏目先做 20 到 50 条精选 Prompt。

推荐栏目：

- 产品广告图
- 人像写真
- 海报设计
- 3D 插画
- 电商主图
- 社媒封面

## 10.2 内容选择原则

第一阶段不追求绝对数量，重点在：

- 案例质量高
- 分类清晰
- 可直接复制
- 可读性强

## 10.3 内容审核标准

发布前至少检查：

- 图片是否足够清晰
- Prompt 是否可读
- 标签是否准确
- 是否重复
- 是否包含高风险内容

## 11. 变现策略

## 11.1 第一阶段可预埋的变现方向

即使 V1 不一定马上收费，也要在结构上为变现预留空间。

优先方向：

- 会员订阅
- Prompt 专题包
- 联盟分成

## 11.2 会员可收费能力

后续适合放到会员的功能包括：

- 更完整的 Prompt 库
- 高级筛选
- 无限复制
- 收藏夹
- Prompt 改写工具
- 批量导出

## 11.3 专题包

可售卖的内容形态：

- 电商专题 Prompt 包
- 海报专题 Prompt 包
- 人像专题 Prompt 包
- 社媒专题 Prompt 包

## 12. 非功能要求

### 12.1 性能

- 首页与列表页应优先保证加载速度
- 图片应支持压缩和多尺寸展示

### 12.2 可维护性

- 数据结构统一
- 页面模块尽量组件化
- 方便后续扩展 Prompt 工具能力

### 12.3 SEO

- 每条 Prompt 应有独立可索引页面
- 页面标题、描述、标签要结构化输出

## 13. 风险与预留

虽然当前先不重点处理来源问题，但系统层面建议预留：

- 来源字段
- 许可字段
- 风险等级字段
- 免责声明页面
- 投诉下架入口

## 14. 技术建议

建议路线：

- 前端：`Next.js`
- 内容存储：本地 JSON / MDX 或轻量数据库
- 图片：对象存储
- 搜索筛选：基于结构化字段实现

第一阶段原则：

- 先跑通内容浏览和复制链路
- 不做过重的后台系统

## 15. 成功标准

如果 V1 达到以下状态，可以认为方向成立：

- 用户能快速理解网站价值
- 用户能在几步内找到想要的 Prompt
- 用户愿意频繁点击复制
- 内容结构支持持续扩充
- 后续收费功能有明确承接点
